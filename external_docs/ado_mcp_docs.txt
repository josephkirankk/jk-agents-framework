### Azure DevOps MCP Server - Wiki Content Management

Source: https://github.com/microsoft/azure-devops-mcp/blob/main/README.md

Details the capabilities for managing wiki content, including creating, updating, and retrieving wiki pages. It emphasizes automatic ETag handling for safe updates.

```APIDOC
wiki_create_page:
  description: Create a new wiki page.
  parameters:
    - name: project
      type: string
      description: The Azure DevOps project name.
    - name: path
      type: string
      description: The path for the new wiki page (e.g., '/Architecture/Overview').
    - name: content
      type: string
      description: The content for the wiki page.
    - name: wikiName
      type: string
      description: The name of the wiki (e.g., 'Documentation'). Defaults to the project's default wiki.
  returns:
    type: object
    description: Details of the created wiki page.

wiki_update_page:
  description: Update an existing wiki page.
  parameters:
    - name: project
      type: string
      description: The Azure DevOps project name.
    - name: path
      type: string
      description: The path of the wiki page to update.
    - name: content
      type: string
      description: The new content for the wiki page.
    - name: etag
      type: string
      description: The ETag of the wiki page for optimistic concurrency control.
    - name: wikiName
      type: string
      description: The name of the wiki. Defaults to the project's default wiki.
  returns:
    type: object
    description: Details of the updated wiki page.

wiki_get_page_content:
  description: Get the content of a specific wiki page.
  parameters:
    - name: project
      type: string
      description: The Azure DevOps project name.
    - name: path
      type: string
      description: The path of the wiki page to retrieve.
    - name: wikiName
      type: string
      description: The name of the wiki. Defaults to the project's default wiki.
  returns:
    type: object
    properties:
      content: string
      etag: string
      path: string
      title: string
      url: string
    description: The content and metadata of the wiki page.
```

### Configure MCP Servers in VS Code for Azure DevOps and Sequential Thinking

      "description": "Azure DevOps organization name  (e.g. 'contoso')"
    }
  ],   
  "servers": {

      "type": "stdio",
      "command": "mcp-server-azuredevops",
      "args": ["${input:ado_org}"]
    },

      "args": ["-y", "@modelcontextprotocol/server-sequential-thinking"]
    }

}

### Troubleshooting Common Naming Issues

Source: https://github.com/microsoft/azure-devops-mcp/blob/main/docs/TOOL-NAME-VALIDATION.md

Provides solutions for common errors encountered during tool development, such as names with spaces, excessive length, or invalid characters.

```text
1. **Tool name contains spaces**
   ```text
   Error: Tool name 'my tool name' contains invalid characters
   ```
   Solution: Use underscores instead: `my_tool_name`

2. **Tool name too long**
   ```text
   Error: Tool name 'very_long_descriptive_tool_name_that_exceeds_limit' is 45 characters long, maximum allowed is 64
   ```
   Solution: Shorten the name: `long_tool_name`

3. **Special characters in name**
   ```text
   Error: Tool name 'tool/name' contains invalid characters
   ```
   Solution: Use allowed characters: `tool_name`
```
