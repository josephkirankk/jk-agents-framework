TITLE: Quick Start Docker Setup for LiteLLM
DESCRIPTION: This snippet outlines the initial steps to get LiteLLM running using Docker. It includes cloning the repository, setting environment variables for master and salt keys, and starting the service with docker-compose.

SOURCE: https://github.com/berriai/litellm/blob/main/docs/my-website/docs/proxy/deploy.md#_snippet_0

LANGUAGE: bash
CODE:
```
git clone https://github.com/BerriAI/litellm

cd litellm

echo 'LITELLM_MASTER_KEY="sk-1234"' > .env

echo 'LITELLM_SALT_KEY="sk-1234"' >> .env

source .env

docker-compose up
```

TITLE: LiteLLM Streaming Usage
DESCRIPTION: Illustrates how to enable streaming for LLM responses using LiteLLM. By setting `stream=True` in the completion arguments, users can receive responses incrementally. This example shows streaming for both OpenAI and Cohere.

SOURCE: https://github.com/berriai/litellm/blob/main/docs/my-website/docs/getting_started.md#_snippet_1

LANGUAGE: python
CODE:
```
from litellm import completion
import os

## set ENV variables

response = completion("command-nightly", messages, stream=True)

```

DESCRIPTION: This bash command starts the LiteLLM proxy using Docker, mounting a local configuration file and setting environment variables for API keys and base URL.

SOURCE: https://github.com/berriai/litellm/blob/main/docs/my-website/docs/proxy/docker_quick_start.md#_snippet_7

LANGUAGE: bash
CODE:
```
docker run \
    -v $(pwd)/litellm_config.yaml:/app/config.yaml \
    -e AZURE_API_KEY=d6*********** \
    -e AZURE_API_BASE=https://openai-***********/ \
    -p 4000:4000 \
    ghcr.io/berriai/litellm:main-latest \
    --config /app/config.yaml --detailed_debug
```
