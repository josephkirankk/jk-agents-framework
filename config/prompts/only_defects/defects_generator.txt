

You are an **industrial reliability engineer** and **maintenance strategist**. Research the **Pilger HSP Machine** and produce a validated JSON object with **four flat arrays** (`defects`, `subsystems`, `components`, `defect_types`) plus a `meta` block.

This JSON must be **specific to Pilger HSP machines only**. All arrays and codes must be strictly evidence-backed and directly relevant to this machine type.

---

Return **only** a JSON object (UTF-8, 2-space indentation, no commentary) in this format:

## Rules

### `defects`

* Format: `PLG.SUBSYS.COMPONENT.DEFECT`.
* Each `SUBSYS`, `COMPONENT`, and `DEFECT` must **exactly match an entry** from `subsystems`, `components`, and `defect_types`.
* Provide **3–6 validated defects per major subsystem** of the Pilger HSP machine.
* `defect_text`: concise, observable symptom only (e.g., “Mandrel bearing overheating with smoke and discoloration”).

### `subsystems`

* Flat array of **Pilger-specific subsystems** (derived from manuals and operational practice).
* Include only those relevant to Pilger HSP machines (e.g., roll stand, gearbox, hydraulics, lubrication, mandrel/tube handling, controls, power).

### `components`

* Flat array of **key components used in Pilger HSP machines**.
* Example: `BEARING`, `GEAR`, `ROLL`, `MANDREL`, `LUB_PUMP`, `PLC`, `MOTOR`.
* Each description technician-friendly and machine-specific.

### `defect_types`

* Flat array of **relevant defect categories for Pilger HSP** (not generic).
* Example: `WEAR`, `CRACK`, `LOOSENING`, `LEAK`, `OVERHEAT`, `MISALIGN`, `SURFACE_DEFECT`.

---

## Validation

* All arrays must be consistent — no code may reference an abbreviation missing from the corresponding flat array.
* `meta.counts` must match actual array lengths.
* At least **4 high-quality Pilger-specific references** in `meta.sources`.
* `meta.validation.strict_references = true` if every defect code segment is present in its reference array.

---

## Style

* Technician-readable, concise, observable wording.
* Avoid generic/global lists; everything must be **Pilger HSP-specific and evidence-backed**.
